# MD2Nuclei

漏洞复现文档批量转换为 Nuclei Templates 的工具。

## 功能特性

- **批量转换**: 支持批量转换整个目录的 Markdown 文档
- **单文件转换**: 支持转换单个 Markdown 文件
- **多线程处理**: 支持指定 worker 数量并行处理，提高转换效率
- **自动分类**: 根据目录结构自动识别漏洞分类（OA、EDU、CMS 等）
- **智能识别**: 自动识别漏洞类型和严重程度
- **FOFA 语法**: 自动提取 FOFA 查询语法
- **失败报告**: 自动生成转换失败报告，方便后续手动处理
- **模板验证**: 支持使用 nuclei 验证生成的模板（需安装 nuclei）

## 项目结构

```
md2nuclei/
├── __init__.py          # 模块初始化
├── main.py              # 主程序入口
├── md_parser.py         # MD文档解析模块
├── http_parser.py       # HTTP请求解析模块
├── nuclei_generator.py  # Nuclei YAML生成模块
└── README.md            # 说明文档
```

## 安装依赖

```bash
pip install pyyaml
```

## 使用方法

### 批量转换

```bash
# 基本用法
python main.py -i /path/to/md/docs -o /path/to/output

# 指定线程数
python main.py -i ./vuln-docs -o ./nuclei-templates -w 8

# 静默模式
python main.py -i ./vuln-docs -o ./nuclei-templates -q
```

### 单文件转换

```bash
python main.py -f /path/to/vuln.md -o /path/to/output.yaml
```

### 验证模板

```bash
# 需要先安装 nuclei
python main.py -f ./vuln.md -o ./output.yaml --validate
```

## 参数说明

| 参数 | 说明 |
|------|------|
| `-i, --input-dir` | 输入目录，包含 Markdown 文件 |
| `-f, --file` | 单个 Markdown 文件路径 |
| `-o, --output` | 输出目录（批量）或文件（单文件） |
| `-w, --workers` | 线程数，默认 4 |
| `-q, --quiet` | 静默模式 |
| `--validate` | 使用 nuclei 验证生成的模板 |

## 支持的文档格式

工具支持以下格式的 Markdown 文档：

```markdown
# 漏洞标题

# 一、漏洞简介
漏洞描述...

# 二、影响版本
* 产品名称/版本

# 三、资产测绘
* fofa`query syntax`

# 四、漏洞复现
```bash
POST /api/endpoint HTTP/1.1
Host: target.com
Header: value

body content
```
```

## 输出示例

### 成功转换的 YAML 模板

```yaml
# Generated by md2nuclei converter
# Date: 2024-01-01 12:00:00

id: 漏洞名称
info:
  name: 漏洞标题
  author: x_x
  severity: high
  description: 漏洞描述
  reference: []
  tags: oa,sqli
  metadata:
    max-request: 1
    fofa-query: body="xxx"
    affected-product: 产品名称
requests:
- method: POST
  path:
  - '{{BaseURL}}/api/endpoint'
  headers:
    Content-Type: application/x-www-form-urlencoded
  body: param=value
  matchers:
  - type: word
    part: body
    words:
    - success
    - true
  - type: status
    status:
    - 200
    - 500
    condition: or
```

转换完成后，会在输出目录生成 `failed_conversion_report.md` 文件：

- `/path/to/failed_file1.md`
- `/path/to/failed_file2.md`
...
```

## 漏洞类型识别

工具会根据漏洞标题和描述自动识别以下漏洞类型：

| 类型 | 关键词 | 严重程度 |
|------|--------|----------|
| RCE | 远程命令执行、远程代码执行、RCE | critical |
| SQL注入 | SQL注入、sqli | high |
| 文件上传 | 文件上传、任意文件上传 | high |
| 文件读取 | 文件读取、任意文件读取 | high |
| 信息泄露 | 信息泄露、信息泄漏、敏感信息 | medium |
| 未授权访问 | 未授权访问 | high |
| SSRF | ssrf | high |
| XXE | xxe | high |

## 注意事项

1. 转换失败的文档通常是因为没有标准的 HTTP 请求格式（如操作系统漏洞、默认口令类漏洞）
2. 生成的模板建议根据实际情况调整 matchers 匹配规则
3. 对于复杂的漏洞场景，建议手动编写模板

## 作为模块使用

```python
from md_parser import MDParser, get_category_from_path
from nuclei_generator import NucleiGenerator

# 解析 MD 文件
parser = MDParser()
vuln_info = parser.parse('/path/to/vuln.md', 'OA')

# 生成 Nuclei 模板
generator = NucleiGenerator()
template = generator.generate(vuln_info)

# 输出 YAML
yaml_content = generator.to_yaml(template)
print(yaml_content)
```
